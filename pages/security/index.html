ওয়েব সিকিউরিটি (Web Security)

পড়তে সময়: ৪৫ মিনিট লেভেল: মধ্যবর্তী থেকে উন্নত

ওয়েব সিকিউরিটি কী?

ওয়েব সিকিউরিটি হলো ওয়েবসাইট এবং ওয়েব অ্যাপ্লিকেশনকে বিভিন্ন সাইবার আক্রমণ থেকে রক্ষা করার প্রক্রিয়া। HTML এবং ব্রাউজার-ভিত্তিক নিরাপত্তা ব্যবস্থা যেমন XSS (ক্রস-সাইট স্ক্রিপ্টিং), ক্লিকজ্যাকিং, এবং তৃতীয় পক্ষের রিসোর্স ম্যানিপুলেশন প্রতিরোধে গুরুত্বপূর্ণ ভূমিকা রাখে। এই অধ্যায়ে আমরা HTML-এর মাধ্যমে প্রয়োগযোগ্য বিভিন্ন নিরাপত্তা ব্যবস্থা শিখব যা আপনার ওয়েবসাইটকে হ্যাকারদের থেকে সুরক্ষিত রাখতে সাহায্য করবে।

শিখবেন যা:

  • XSS (ক্রস-সাইট স্ক্রিপ্টিং): ইনজেকশন আক্রমণ কীভাবে প্রতিরোধ করবেন
  • CSP (কন্টেন্ট সিকিউরিটি পলিসি): ব্রাউজারকে কোন রিসোর্স লোড করতে পারবে তা নিয়ন্ত্রণ
  • স্যান্ডবক্স: iframe-এ সীমাবদ্ধ পরিবেশ তৈরি
  • SRI (সাবরিসোর্স ইন্টিগ্রিটি): CDN থেকে লোড করা ফাইলের অখণ্ডতা যাচাই
  • ক্লিকজ্যাকিং প্রতিরোধ: X-Frame-Options এবং frame-ancestors ব্যবহার

XSS প্রতিরোধ (Cross-Site Scripting Prevention)

XSS (ক্রস-সাইট স্ক্রিপ্টিং) হলো সবচেয়ে সাধারণ ওয়েব নিরাপত্তা দুর্বলতা যেখানে আক্রমণকারী ওয়েবসাইটে ম্যালিশিয়াস জাভাস্ক্রিপ্ট ইনজেক্ট করে। HTML-এ সঠিকভাবে কন্টেন্ট এস্কেপিং এবং নিরাপদ অ্যাট্রিবিউট ব্যবহার করে XSS প্রতিরোধ করা যায়।

XSS এর উদাহরণ (কীভাবে কাজ করে):

<!-- আক্রমণকারী ইনপুট ফিল্ডে এই কোড দিতে পারে -->
<script>alert('XSS আক্রমণ!');</script>
<img src=x onerror=alert('XSS')>

<!-- HTML এস্কেপিং না করলে -->
<div> <script>alert('XSS')</script> </div> <!-- বিপজ্জনক -->

<!-- সঠিক এস্কেপিং (টেক্সট হিসেবে দেখাবে) -->
<div> &lt;script&gt;alert('XSS')&lt;/script&gt; </div>

XSS প্রতিরোধের উপায়:

  • HTML এস্কেপিং: < → &lt;, > → &gt;, & → &amp;, " → &quot;
  • HTTP-only Cookies: Set-Cookie: session=xyz; HttpOnly (JavaScript-এ cookie অ্যাক্সেস রোধ)
  • CSP (Content Security Policy): inline script বন্ধ করা
  • ব্যাকএন্ডে ইনপুট ভ্যালিডেশন ও স্যানিটাইজেশন
  • innerHTML এর পরিবর্তে textContent ব্যবহার

নিরাপদ DOM ম্যানিপুলেশন:

// বিপজ্জনক - XSS ঝুঁকি
element.innerHTML = userInput;

// নিরাপদ - টেক্সট হিসেবে ইনসার্ট
element.textContent = userInput;

// নিরাপদ attribute সেট
element.setAttribute('data-value', userInput);
img.src = validatedUrl; // URL ভ্যালিডেশন প্রয়োজন

কন্টেন্ট সিকিউরিটি পলিসি (CSP - Content Security Policy)

CSP একটি শক্তিশালী নিরাপত্তা ব্যবস্থা যা ব্রাউজারকে বলে কোন উৎস থেকে কোন ধরনের কন্টেন্ট লোড করা যাবে। এটি XSS, 데이터 ইনজেকশন এবং অন্যান্য আক্রমণ প্রতিরোধে কার্যকর।

CSP হেডার উদাহরণ:

<!-- HTTP হেডার হিসেবে (সবচেয়ে নিরাপদ) -->
Content-Security-Policy: default-src 'self'; script-src 'self' https://trusted.cdn.com; style-src 'self' 'unsafe-inline'; img-src *; object-src 'none'; base-uri 'self';

<!-- HTML মেটা ট্যাগ (সীমিত সাপোর্ট) -->
<meta http-equiv="Content-Security-Policy" content="default-src 'self'; script-src 'self'">

CSP ডাইরেক্টিভ সমূহ:

  • default-src 'self' – ডিফল্ট সোর্স নিজের ডোমেইন
  • script-src 'self' – JavaScript শুধু নিজের ডোমেইন থেকে
  • style-src 'self' 'unsafe-inline' – CSS নিজের ডোমেইন + inline স্টাইল
  • img-src * – ইমেজ যেকোনো জায়গা থেকে
  • font-src https://fonts.googleapis.com – ফন্ট নির্দিষ্ট CDN থেকে
  • object-src 'none' – 플러그িন (Flash, Java) বন্ধ
  • frame-ancestors 'none' – ক্লিকজ্যাকিং প্রতিরোধ
  • report-uri /csp-report – লঙ্ঘন রিপোর্ট পাঠানোর ঠিকানা

CSP রিপোর্টিং (Content-Security-Policy-Report-Only):

<!-- শুধু রিপোর্ট করবে, ব্লক করবে না -->
Content-Security-Policy-Report-Only: default-src 'self'; report-uri /csp-violation

// লঙ্ঘন রিপোর্ট JSON ফরম্যাটে আসে
{
  "csp-report": {
    "document-uri": "https://example.com/page",
    "violated-directive": "script-src 'self'",
    "blocked-uri": "https://evil.com/script.js"
  }
}

প্র্যাকটিস: প্রথমে Report-Only মোডে চালিয়ে লঙ্ঘন চিহ্নিত করুন, তারপর পূর্ণ CSP প্রয়োগ করুন।

স্যান্ডবক্স (Sandbox)

sandbox অ্যাট্রিবিউট iframe-এর মধ্যে চলমান কন্টেন্টের ক্ষমতা সীমিত করে। এটি সন্দেহজনক বা থার্ড-পার্টি কন্টেন্ট নিরাপদে এম্বেড করতে ব্যবহৃত হয়।

iframe স্যান্ডবক্স উদাহরণ:

<!-- সম্পূর্ণ সীমাবদ্ধ (সবচেয়ে নিরাপদ) -->
<iframe src="https://untrusted.com" sandbox></iframe>

<!-- নির্দিষ্ট পারমিশন সহ -->
<iframe src="https://untrusted.com" 
        sandbox="allow-same-origin allow-scripts allow-forms allow-popups">
</iframe>

<!-- allow-same-origin ছাড়া allow-scripts দিলে স্ক্রিপ্ট কাজ করে না -->

স্যান্ডবক্স পারমিশন:

  • allow-same-origin – একই অরিজিনের রিসোর্স অ্যাক্সেস
  • allow-scripts – JavaScript চালানোর অনুমতি
  • allow-forms – ফর্ম সাবমিট করার অনুমতি
  • allow-popups – পপআপ উইন্ডো খোলার অনুমতি
  • allow-top-navigation – top-level ব্রাউজিং কন্টেক্সট নেভিগেট করার অনুমতি
  • allow-modals – মোডাল ডায়ালগ (alert, confirm) দেখানোর অনুমতি
  • allow-presentation – presentation session শুরু করার অনুমতি

স্যান্ডবক্স ছাড়া vs স্যান্ডবক্স সহ:

<!-- বিপজ্জনক: iframe-এর স্ক্রিপ্ট প্যারেন্ট পেজে অ্যাক্সেস পেতে পারে -->
<iframe src="malicious.html"></iframe>

<!-- নিরাপদ: স্যান্ডবক্স স্ক্রিপ্ট, ফর্ম, নেভিগেশন ব্লক করে -->
<iframe src="untrusted.html" sandbox></iframe>

<!-- নির্দিষ্ট অনুমতি দিয়ে -->
<iframe src="widget.html" sandbox="allow-same-origin allow-scripts"></iframe>

সাবরিসোর্স ইন্টিগ্রিটি (SRI - Subresource Integrity)

SRI ব্রাউজারকে CDN বা থার্ড-পার্টি থেকে লোড করা ফাইলের ক্রিপ্টোগ্রাফিক হ্যাশ যাচাই করতে দেয়। ফাইল পরিবর্তিত হলে ব্রাউজার তা লোড করবে না। এটি CDN হ্যাক বা ম্যান-ইন-দ্য-মিডল আক্রমণ প্রতিরোধ করে।

SRI উদাহরণ:

<!-- SRI সহ CSS ফাইল -->
<link rel="stylesheet" 
      href="https://cdn.example.com/style.css" 
      integrity="sha384-oqVuAfXRKap7fdgcCY5uykM6+R9GqQ8K/uxy9rx7HNQlGYl1kPzQho1wx4JwY8wC" 
      crossorigin="anonymous">

<!-- SRI সহ JavaScript ফাইল -->
<script src="https://cdn.example.com/app.js" 
        integrity="sha384-Li9vy3DqF8tnTXuiaAJuML3ky+er10rcgNR/VqsVpcw+ThHmYcwiB1pbOxEbzJr7" 
        crossorigin="anonymous"></script>

SRI হ্যাশ জেনারেট করুন: openssl dgst -sha384 -binary filename.js | openssl base64 -A অথবা srihash.org ব্যবহার করুন।

SRI কিভাবে কাজ করে:

  • ব্রাউজার ফাইল ডাউনলোড করে
  • হ্যাশ গণনা করে এবং integrity অ্যাট্রিবিউটের সাথে তুলনা করে
  • মিল না-মিললে ফাইল লোড হয় না এবং কনসোলে এরর দেখায়
  • crossorigin অ্যাট্রিবিউট CORS-enabled রিসোর্সের জন্য প্রয়োজন

ক্লিকজ্যাকিং প্রতিরোধ (Clickjacking Prevention)

ক্লিকজ্যাকিং একটি আক্রমণ যেখানে আক্রমণকারী স্বচ্ছ iframe ব্যবহার করে ব্যবহারকারীকে অজান্তে অন্য সাইটে ক্লিক করতে বাধ্য করে। X-Frame-Options হেডার বা CSP-র frame-ancestors ব্যবহার করে এটি প্রতিরোধ করা যায়।

ক্লিকজ্যাকিং প্রতিরোধের পদ্ধতি:

<!-- HTTP হেডার (সবচেয়ে সহজ) -->
X-Frame-Options: DENY
X-Frame-Options: SAMEORIGIN

<!-- CSP frame-ancestors (আধুনিক, আরও নিয়ন্ত্রণ) -->
Content-Security-Policy: frame-ancestors 'none';
Content-Security-Policy: frame-ancestors 'self' https://trusted.com;

<!-- HTML মেটা ট্যাগ (সীমিত কার্যকারিতা) -->
<meta http-equiv="X-Frame-Options" content="DENY">

X-Frame-Options মান:

  • DENY – কোনো iframe-এ এম্বেড করা যাবে না
  • SAMEORIGIN – শুধু একই অরিজিনের পৃষ্ঠায় এম্বেড করা যাবে
  • ALLOW-FROM uri – নির্দিষ্ট URI-তে অনুমতি (পুরনো, CSP ব্যবহার ভালো)

JavaScript দিয়ে ফ্রেম-বাস্টিং (অতিরিক্ত সুরক্ষা):

<script>
// ফ্রেম-বাস্টিং - iframe-এ থাকলে টপে রিডাইরেক্ট
if (window.top !== window.self) {
  window.top.location = window.self.location;
}
</script>

সেরা প্র্যাকটিস: HTTP হেডার + CSP একসাথে ব্যবহার করুন। JavaScript ফ্রেম-বাস্টিং শুধু অতিরিক্ত সুরক্ষা হিসেবে রাখুন।

অতিরিক্ত HTML নিরাপত্তা টিপস

নিরাপদ HTML অনুশীলন:

  • HTTPS ব্যবহার করুন: সব পৃষ্ঠায় HTTPS বাধ্যতামূলক করুন, HSTS (Strict-Transport-Security) হেডার যোগ করুন
  • Referrer Policy: <meta name="referrer" content="no-referrer"> দিয়ে referrer তথ্য নিয়ন্ত্রণ করুন
  • নিরাপদ cookie অ্যাট্রিবিউট: Secure; HttpOnly; SameSite=Strict
  • X-Content-Type-Options: X-Content-Type-Options: nosniff – MIME sniffing প্রতিরোধ
  • input সীমাবদ্ধতা: input এ type="email", pattern, maxlength ব্যবহার করুন

নিরাপদ HTML হেডার (Apache/NGINX কনফিগ):

# নিরাপত্তা হেডার
Header set X-Frame-Options "DENY"
Header set X-Content-Type-Options "nosniff"
Header set X-XSS-Protection "1; mode=block"
Header set Referrer-Policy "strict-origin-when-cross-origin"
Header set Content-Security-Policy "default-src 'self'; script-src 'self'; style-src 'self'; img-src 'self' data:;"

# HSTS (HTTPS বাধ্যতামূলক)
Header set Strict-Transport-Security "max-age=31536000; includeSubDomains; preload"

সিকিউরিটি চেকলিস্ট

HTML ওয়েবসাইট সিকিউরিটি চেকলিস্ট:

  • ✅ HTTPS ব্যবহার করুন (SSL/TLS সার্টিফিকেট)
  • ✅ CSP (Content Security Policy) কনফিগার করুন
  • ✅ X-Frame-Options: DENY বা SAMEORIGIN সেট করুন
  • ✅ X-Content-Type-Options: nosniff সেট করুন
  • ✅ থার্ড-পার্টি স্ক্রিপ্টে SRI ব্যবহার করুন
  • ✅ iframe-এ sandbox অ্যাট্রিবিউট ব্যবহার করুন
  • ✅ cookies-এ Secure, HttpOnly, SameSite ফ্ল্যাগ ব্যবহার করুন
  • ✅ ইনপুট ডাটা HTML এস্কেপ করুন
  • ✅ user-generated HTML এ DOMPurify বা অনুরূপ লাইব্রেরি ব্যবহার করুন
  • ✅ নিরাপত্তা হেডার পরীক্ষার জন্য securityheaders.com ব্যবহার করুন

নিরাপত্তা টুলস ও রিসোর্স

# নিরাপত্তা হেডার পরীক্ষার টুলস
# https://securityheaders.com
# https://observatory.mozilla.org

# CSP পরীক্ষার টুল
# https://csp-evaluator.withgoogle.com

# SRI হ্যাশ জেনারেটর
# https://www.srihash.org
# openssl dgst -sha384 -binary script.js | openssl base64 -A

সম্পূর্ণ টপিক লিস্ট

নিচের লিংকগুলোতে ক্লিক করে প্রতিটি টপিক বিস্তারিত শিখুন:
