ওয়েব পারফরম্যান্স (Web Performance)

পড়তে সময়: ৫০ মিনিট লেভেল: মধ্যবর্তী থেকে উন্নত

ওয়েব পারফরম্যান্স কী?

ওয়েব পারফরম্যান্স হলো ওয়েবসাইটের লোডিং স্পিড, রেসপন্সিভনেস এবং ভিজ্যুয়াল স্থিতিশীলতা পরিমাপ ও উন্নত করার প্রক্রিয়া। দ্রুত লোড হওয়া ওয়েবসাইট শুধু ইউজার এক্সপেরিয়েন্সই উন্নত করে না, বরং SEO র্যাঙ্কিং, কনভার্সন রেট এবং ব্যবহারকারীর ধারণার উপরও বিশাল প্রভাব ফেলে। Google-এর গবেষণা অনুযায়ী, ৩ সেকেন্ডের বেশি লোড হতে থাকলে ৫৩% মোবাইল ব্যবহারকারী সাইট ছেড়ে চলে যায়। এই অধ্যায়ে আমরা HTML, CSS, JavaScript এবং রিসোর্স অপটিমাইজেশনের মাধ্যমে ওয়েব পারফরম্যান্স উন্নত করার কৌশল শিখব।

শিখবেন যা:

  • পেজ স্পিড মেট্রিক্স: FCP, LCP, TTI, CLS বুঝতে শেখা
  • রিসোর্স অপটিমাইজেশন: লেজি লোডিং, প্রিলোড, প্রিফেচ, অ্যাসিঙ্ক/ডিফার
  • ইমেজ অপটিমাইজেশন: ফরম্যাট নির্বাচন, কম্প্রেশন, responsive images
  • ক্যাশিং স্ট্রাটেজি: ব্রাউজার ক্যাশ, HTTP ক্যাশ হেডার
  • ক্রিটিকাল রেন্ডারিং পাথ: রেন্ডার-ব্লকিং রিসোর্স কমানো
  • নেটওয়ার্ক অপটিমাইজেশন: preconnect, dns-prefetch

পেজ স্পিড (Page Speed) ও Core Web Vitals

পেজ স্পিড পরিমাপের জন্য Google-এর Core Web Vitals তিনটি গুরুত্বপূর্ণ মেট্রিক নির্ধারণ করেছে যা ইউজার এক্সপেরিয়েন্স পরিমাপ করে।

Core Web Vitals মেট্রিক্স:

  • LCP (Largest Contentful Paint): পৃষ্ঠার প্রধান কন্টেন্ট লোড হতে সময়। গুড: ≤ 2.5 সেকেন্ড
  • FID (First Input Delay): ব্যবহারকারীর প্রথম ইন্টারঅ্যাকশনে ব্রাউজারের রেসপন্স সময়। গুড: ≤ 100ms
  • CLS (Cumulative Layout Shift): পৃষ্ঠা লোডের সময় লেআউটের স্থিতিশীলতা। গুড: ≤ 0.1

পারফরম্যান্স মনিটরিং টুলস:

// Lighthouse in Chrome DevTools
// PageSpeed Insights: https://pagespeed.web.dev
// Web Vitals JavaScript লাইব্রেরি
import {getLCP, getFID, getCLS} from 'web-vitals';

getLCP(console.log);
getFID(console.log);
getCLS(console.log);

টিপ: Chrome DevTools-এর Performance ট্যাব ব্যবহার করে রিয়েল-টাইমে আপনার সাইটের পারফরম্যান্স বিশ্লেষণ করুন।

লেজি লোডিং (Lazy Loading)

লেজি লোডিং এমন একটি কৌশল যেখানে ইমেজ, iframe, বা ভিডিও শুধুমাত্র তখনই লোড হয় যখন তা ব্যবহারকারীর ভিউপোর্টে আসে। এটি প্রাথমিক পৃষ্ঠা লোডের সময় এবং ডাটা খরচ কমায়।

নেটিভ লেজি লোডিং:

<!-- ইমেজ লেজি লোডিং -->
<img src="image.jpg" loading="lazy" alt="লেজি লোড ইমেজ">

<!-- iframe লেজি লোডিং -->
<iframe src="video.html" loading="lazy"></iframe>

<!-- কাস্টম JavaScript লেজি লোডিং (Intersection Observer) -->
<img data-src="image.jpg" class="lazy" alt="লেজি লোড">

<script>
const lazyImages = document.querySelectorAll('.lazy');
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      const img = entry.target;
      img.src = img.dataset.src;
      observer.unobserve(img);
    }
  });
});
lazyImages.forEach(img => observer.observe(img));
</script>

ব্রাউজার সাপোর্ট: loading="lazy" আধুনিক সব ব্রাউজারে সাপোর্টেড। পুরনো ব্রাউজারের জন্য Intersection Observer ব্যবহার করুন।

প্রিলোড ও প্রিফেচ (Preload & Prefetch)

preload এবং prefetch রিসোর্স লোডিং অগ্রাধিকার নিয়ন্ত্রণ করে। preload গুরুত্বপূর্ণ রিসোর্স তাড়াতাড়ি লোড করে, prefetch ভবিষ্যতে প্রয়োজন হতে পারে এমন রিসোর্স ব্যাকগ্রাউন্ডে লোড করে।

প্রিলোড ও প্রিফেচ উদাহরণ:

<!-- প্রিলোড: গুরুত্বপূর্ণ রিসোর্স তাড়াতাড়ি লোড করুন -->
<link rel="preload" href="critical-font.woff2" as="font" type="font/woff2" crossorigin>
<link rel="preload" href="hero-image.jpg" as="image">
<link rel="preload" href="style.css" as="style">

<!-- প্রিফেচ: পরবর্তী পৃষ্ঠার জন্য রিসোর্স প্রি-লোড -->
<link rel="prefetch" href="next-page.html">
<link rel="prefetch" href="next-page-script.js">

<!-- মডিউল প্রিলোড -->
<link rel="modulepreload" href="app.js">

কখন ব্যবহার করবেন:

  • preload: LCP ইমেজ, ক্রিটিকাল CSS, ফন্ট, গুরুত্বপূর্ণ স্ক্রিপ্ট
  • prefetch: পরবর্তী পৃষ্ঠার রিসোর্স, নেভিগেশন প্রেডিকশন

অ্যাসিঙ্ক ও ডিফার (Async vs Defer)

async এবং defer অ্যাট্রিবিউট JavaScript লোডিং পদ্ধতি নিয়ন্ত্রণ করে। এগুলি রেন্ডার-ব্লকিং প্রতিরোধ করে পৃষ্ঠা লোড দ্রুত করে।

<!-- ডিফল্ট (রেন্ডার-ব্লকিং) -->
<script src="script.js"></script>

<!-- অ্যাসিঙ্ক: প্যারালাল ডাউনলোড, ডাউনলোড শেষে সাথে সাথে এক্সিকিউট -->
<script async src="analytics.js"></script>

<!-- ডিফার: প্যারালাল ডাউনলোড, HTML পার্স শেষে এক্সিকিউট -->
<script defer src="app.js"></script>
অ্যাট্রিবিউটডাউনলোডএক্সিকিউশন টাইমঅর্ডার নিশ্চিত?
ডিফল্টHTML পার্স ব্লক করেডাউনলোড শেষে সাথেহ্যাঁ
asyncপ্যারালালডাউনলোড শেষে সাথে (HTML পার্স পজ)না
deferপ্যারালালHTML পার্স শেষেহ্যাঁ

বেস্ট প্র্যাকটিস: থার্ড-পার্টি স্ক্রিপ্ট (analytics) জন্য async, অ্যাপ্লিকেশন স্ক্রিপ্টের জন্য defer, এবং ক্রিটিকাল স্ক্রিপ্টের জন্য async অথবা inline ব্যবহার করুন।

ইমেজ অপটিমাইজেশন (Image Optimization)

ইমেজ সাধারণত ওয়েবপৃষ্ঠার সবচেয়ে বড় রিসোর্স। সঠিক ফরম্যাট নির্বাচন, কম্প্রেশন, এবং responsive images ব্যবহার করে পারফরম্যান্স ব্যাপকভাবে উন্নত করা যায়।

ইমেজ অপটিমাইজেশন টেকনিক:

<!-- আধুনিক ফরম্যাট (WebP/AVIF) -->
<picture>
  <source srcset="image.avif" type="image/avif">
  <source srcset="image.webp" type="image/webp">
  <img src="image.jpg" alt="অপটিমাইজড ইমেজ">
</picture>

<!-- responsive images with srcset -->
<img src="small.jpg"
     srcset="small.jpg 480w,
             medium.jpg 800w,
             large.jpg 1200w"
     sizes="(max-width: 600px) 480px,
            (max-width: 1000px) 800px,
            1200px"
     alt="রেসপনসিভ ইমেজ">

<!-- width/height অ্যাট্রিবিউট CLS প্রতিরোধে -->
<img src="image.jpg" width="800" height="600" alt="ক্লিএস সুরক্ষিত">

ইমেজ অপটিমাইজেশন চেকলিস্ট:

  • WebP বা AVIF ফরম্যাট ব্যবহার করুন
  • ইমেজ কম্প্রেস করুন (TinyPNG, Squoosh, ImageOptim)
  • loading="lazy" ব্যবহার করুন
  • width ও height অ্যাট্রিবিউট দিন (CLS প্রতিরোধ)
  • responsive images-এর জন্য srcset ও sizes ব্যবহার করুন

ক্যাশিং (Caching)

ক্যাশিং ব্রাউজারে রিসোর্স সংরক্ষণ করে পরবর্তী ভিজিটে দ্রুত লোড নিশ্চিত করে। HTTP ক্যাশ হেডার এবং Service Workers এর মাধ্যমে ক্যাশিং নিয়ন্ত্রণ করা যায়।

HTTP ক্যাশ হেডার (Server-side):

# .htaccess (Apache) বা server config
<FilesMatch "\.(css|js|jpg|png|webp|woff2)$">
  Header set Cache-Control "max-age=31536000, public, immutable"
</FilesMatch>

<FilesMatch "\.(html|php)$">
  Header set Cache-Control "max-age=3600, must-revalidate"
</FilesMatch>

# HTML meta tag (limited)
<meta http-equiv="Cache-Control" content="no-cache">

ক্যাশিং স্ট্রাটেজি: স্ট্যাটিক রিসোর্স (CSS, JS, ইমেজ) জন্য long-term cache (1 year), HTML ফাইলের জন্য short-term cache (1 hour)। ফাইল পরিবর্তনে ক্যাশ ইনভ্যালিডেশনের জন্য ফাইলে hash ব্যবহার করুন (style.abc123.css)।

ক্রিটিকাল রেন্ডারিং পাথ (Critical Rendering Path)

ক্রিটিকাল রেন্ডারিং পাথ হলো ব্রাউজার HTML, CSS, JavaScript প্রক্রিয়া করে প্রথম পেইন্ট তৈরি করার প্রক্রিয়া। রেন্ডার-ব্লকিং রিসোর্স কমানো এই পাথকে অপটিমাইজ করে।

CRP অপটিমাইজেশন টেকনিক:

<!-- ক্রিটিকাল CSS inline করুন -->
<style>
  /* above-the-fold কন্টেন্টের জন্য প্রয়োজনীয় CSS */
  body { margin: 0; font-family: sans-serif; }
  .header { background: #333; color: white; }
</style>

<!-- নন-ক্রিটিকাল CSS defer বা preload -->
<link rel="preload" href="full-styles.css" as="style" onload="this.onload=null;this.rel='stylesheet'">
<noscript><link rel="stylesheet" href="full-styles.css"></noscript>

<!-- JavaScript defer -->
<script defer src="app.js"></script>

<!-- প্রিলোড গুরুত্বপূর্ণ ফন্ট -->
<link rel="preload" href="font.woff2" as="font" type="font/woff2" crossorigin>

রেন্ডার-ব্লকিং রিসোর্স চিহ্নিত করুন:

  • CSS ডিফল্টভাবে রেন্ডার-ব্লকিং করে (ক্রিটিকাল CSS inline করুন)
  • JavaScript (defer/async ব্যবহার করুন)
  • ফন্ট (preload ব্যবহার করুন)

প্রিকানেক্ট ও ডিএনএস-প্রিফেচ (Preconnect & DNS-Prefetch)

প্রিকানেক্ট এবং DNS-প্রিফেচ থার্ড-পার্টি রিসোর্স (API, ফন্ট, CDN) এর জন্য সংযোগ স্থাপনের সময় কমায়।

<!-- DNS প্রিফেচ: DNS lookup আগেই করে রাখে -->
<link rel="dns-prefetch" href="https://fonts.googleapis.com">
<link rel="dns-prefetch" href="https://analytics.google.com">

<!-- প্রিকানেক্ট: DNS lookup + TCP handshake + TLS negotiation -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>

<!-- থার্ড-পার্টি রিসোর্সের জন্য ব্যবহার করুন -->
<link rel="preconnect" href="https://api.example.com">
<link rel="preconnect" href="https://cdn.example.com">

কখন ব্যবহার করবেন: প্রিকানেক্ট শুধুমাত্র গুরুত্বপূর্ণ থার্ড-পার্টি অরিজিনের জন্য ব্যবহার করুন (অতিরিক্ত ব্যবহারে ব্যাকফায়ার হতে পারে)। DNS-প্রিফেচ বেশি নিরাপদ এবং সব অরিজিনের জন্য ব্যবহারযোগ্য।

পারফরম্যান্স বেস্ট প্র্যাকটিস চেকলিস্ট

HTML পারফরম্যান্স চেকলিস্ট:

  • ✅ <link rel="preload"> দিয়ে ক্রিটিকাল রিসোর্স প্রিলোড করুন
  • ✅ ইমেজে loading="lazy" ব্যবহার করুন
  • ✅ স্ক্রিপ্টে defer বা async ব্যবহার করুন
  • ✅ ক্রিটিকাল CSS inline করুন, বাকি CSS defer করুন
  • ✅ ইমেজের জন্য width ও height নির্ধারণ করুন (CLS প্রতিরোধ)
  • ✅ WebP/AVIF ফরম্যাট ব্যবহার করুন
  • ✅ থার্ড-পার্টি স্ক্রিপ্টে async ব্যবহার করুন
  • ✅ প্রিকানেক্ট ও DNS-প্রিফেচ ব্যবহার করুন
  • ✅ ক্যাশ হেডার সঠিকভাবে কনফিগার করুন
  • ✅ Core Web Vitals মনিটর করুন
