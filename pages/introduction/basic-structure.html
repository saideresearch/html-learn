HTML মৌলিক কাঠামো (Basic Structure of HTML)

পড়তে সময়: ১২ মিনিট লেভেল: শিক্ষানবিশ

HTML ডকুমেন্টের গঠন

প্রতিটি HTML ডকুমেন্টের একটি নির্দিষ্ট কাঠামো থাকে। এই কাঠামো ব্রাউজারকে বুঝতে সাহায্য করে যে এটি একটি HTML ডকুমেন্ট এবং কীভাবে তা প্রদর্শন করতে হবে। নিচে একটি সাধারণ HTML ডকুমেন্টের কাঠামো দেওয়া হলো:

<!DOCTYPE html>
<html>
  <head>
    <title>পৃষ্ঠার শিরোনাম</title>
  </head>
  <body>
    <h1>এটি একটি শিরোনাম</h1>
    <p>এটি একটি প্যারাগ্রাফ</p>
  </body>
</html>

কাঠামোর চিত্র:

  ├─ <!DOCTYPE html>
  └─ <html>
      ├─ <head>
      │    ├─ <title>পৃষ্ঠার শিরোনাম</title>
      │    └─ অন্যান্য মেটাডাটা
      └─ <body>
           ├─ <h1>শিরোনাম</h1>
           └─ <p>প্যারাগ্রাফ</p>

১. <!DOCTYPE> ডিক্লারেশন

<!DOCTYPE> ডিক্লারেশন HTML ডকুমেন্টের একদম শুরুতে থাকে। এটি ব্রাউজারকে জানায় যে এই ডকুমেন্টটি কোন HTML সংস্করণে লেখা হয়েছে।

<!DOCTYPE html>

বৈশিষ্ট্য:

  • এটি একটি ট্যাগ নয়, এটি একটি ডিক্লারেশন মাত্র
  • এটি কেস-সংবেদনশীল নয় (ছোট বা বড় হাতের অক্ষর দুটোই চলে)
  • এটি HTML5-এর জন্য সবচেয়ে সহজ ও সংক্ষিপ্ত
  • এটি অবশ্যই ডকুমেন্টের প্রথম লাইনে থাকতে হবে

পুরনো HTML সংস্করণের জন্য DOCTYPE:

HTML সংস্করণ DOCTYPE ডিক্লারেশন
HTML 4.01 Strict <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">
HTML 4.01 Transitional <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">
XHTML 1.0 Strict <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">

💡 মজার তথ্য: HTML5-এর DOCTYPE সবচেয়ে ছোট এবং সহজ। এটি ডিজাইন করা হয়েছে যাতে ডেভেলপাররা সহজেই মনে রাখতে পারেন।

২. <html> এলিমেন্ট

<html> এলিমেন্ট হলো HTML ডকুমেন্টের মূল বা রুট এলিমেন্ট। এই এলিমেন্টের ভিতরেই ডকুমেন্টের বাকি সব এলিমেন্ট থাকে।

<html lang="bn">
  ... পুরো ডকুমেন্ট ...
</html>

গুরুত্বপূর্ণ অ্যাট্রিবিউট:

অ্যাট্রিবিউট বর্ণনা উদাহরণ
lang ডকুমেন্টের ভাষা নির্ধারণ করে lang="bn" (বাংলা), lang="en" (ইংরেজি)
dir টেক্সটের দিক নির্ধারণ করে dir="ltr" (বাম থেকে ডান), dir="rtl" (ডান থেকে বাম)

lang="bn" অ্যাট্রিবিউট যোগ করলে ব্রাউজার বুঝতে পারে যে এই পৃষ্ঠাটি বাংলা ভাষায় লেখা, যা সার্চ ইঞ্জিন ও স্ক্রিন রিডারের জন্য উপকারী।

৩. <head> এলিমেন্ট

<head> এলিমেন্টটি ডকুমেন্ট সম্পর্কে তথ্য (মেটাডাটা) ধারণ করে। এই এলিমেন্টের ভিতরে যা লেখা হয়, তা ব্রাউজারের মূল পৃষ্ঠায় দেখা যায় না।

<head>
  <meta charset="UTF-8">
  <title>আমার ওয়েবপৃষ্ঠা</title>
  <link rel="stylesheet" href="style.css">
  <meta name="description" content="এটি একটি শিক্ষামূলক ওয়েবসাইট">
  <meta name="keywords" content="HTML, শিক্ষা, ওয়েব ডেভেলপমেন্ট">
  <meta name="author" content="জন দো">
</head>

<head> এলিমেন্টের সাধারণ এলিমেন্টসমূহ:

এলিমেন্ট কাজ উদাহরণ
<title> ব্রাউজারের ট্যাবে শিরোনাম দেখায় <title>হোম পৃষ্ঠা</title>
<meta> মেটাডাটা প্রদান করে (চারসেট, বর্ণনা, কীওয়ার্ড ইত্যাদি) <meta charset="UTF-8">
<link> বাহ্যিক রিসোর্স লিংক করে (সিএসএস, আইকন) <link rel="stylesheet" href="style.css">
<style> অভ্যন্তরীণ সিএসএস লেখার জন্য <style>body {color: black;}</style>
<script> জাভাস্ক্রিপ্ট কোড যুক্ত করার জন্য <script src="script.js"></script>

৪. <body> এলিমেন্ট

<body> এলিমেন্টের ভিতরে ডকুমেন্টের মূল কন্টেন্ট লেখা হয় যা ব্রাউজারে প্রদর্শিত হয়।

<body>
  <h1>প্রধান শিরোনাম</h1>
  <p>এটি একটি প্যারাগ্রাফ যা ব্রাউজারে দেখা যাবে।</p>
  <img src="ছবি.jpg" alt="ছবির বর্ণনা">
  <a href="https://example.com">একটি লিংক</a>
</body>

<body> এলিমেন্টের সাধারণ এলিমেন্টসমূহ:

  • শিরোনাম: <h1> থেকে <h6>
  • প্যারাগ্রাফ: <p>
  • লিংক: <a>
  • ছবি: <img>
  • তালিকা: <ul>, <ol>, <li>
  • টেবিল: <table>
  • ফর্ম: <form>, <input>
  • ডিভিশন: <div>
  • স্প্যান: <span>

সম্পূর্ণ মৌলিক কাঠামো (বাংলা সংস্করণ)

বাংলা ভাষায় একটি সম্পূর্ণ HTML পৃষ্ঠার মৌলিক কাঠামো নিচে দেওয়া হলো:

<!DOCTYPE html>
<html lang="bn">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>বাংলায় HTML পৃষ্ঠা</title>
  <meta name="description" content="এটি একটি বাংলা HTML পৃষ্ঠার উদাহরণ">
  <meta name="keywords" content="HTML, বাংলা, শিক্ষা">
  <meta name="author" content="HTML শিখি">
</head>
<body>
  <h1>বাংলায় স্বাগতম!</h1>
  <p>এটি একটি সম্পূর্ণ বাংলা HTML পৃষ্ঠা। এখানে আমরা HTML-এর মৌলিক কাঠামো শিখছি।</p>
  
  <h2>HTML-এর মূল অংশগুলো:</h2>
  <ul>
    <li>DOCTYPE ডিক্লারেশন</li>
    <li>html এলিমেন্ট</li>
    <li>head এলিমেন্ট (মেটাডাটা)</li>
    <li>body এলিমেন্ট (দৃশ্যমান কন্টেন্ট)</li>
  </ul>
  
  <p>আশা করি আপনি HTML শেখা উপভোগ করছেন। <a href="#">আরও পড়ুন</a></p>
</body>
</html>

HTML কাঠামোর ভিজুয়াল রিপ্রেজেন্টেশন

বাংলায় HTML পৃষ্ঠা
<body>
<h1> বাংলায় স্বাগতম! </h1>
<p> এটি একটি সম্পূর্ণ বাংলা HTML পৃষ্ঠা। </p>
<ul>
<li>DOCTYPE ডিক্লারেশন</li>
<li>html এলিমেন্ট</li>
<li>head এলিমেন্ট</li>
<li>body এলিমেন্ট</li>
</ul>
</body>

HTML এলিমেন্টের নেস্টিং (Nesting)

HTML-এ এক এলিমেন্টের ভিতরে আরেক এলিমেন্ট রাখাকে নেস্টিং বলে। সঠিক নেস্টিং খুবই গুরুত্বপূর্ণ।

✅ সঠিক নেস্টিং:

<div>
  <p>এটি একটি <strong>গুরুত্বপূর্ণ</strong> প্যারাগ্রাফ</p>
</div>

❌ ভুল নেস্টিং:

<div>
  <p>এটি একটি <strong>গুরুত্বপূর্ণ</div> প্যারাগ্রাফ</strong>
</p>

ভুল নেস্টিং-এ ট্যাগের ক্রম ঠিক রাখা হয়নি। মনে রাখবেন: যে ট্যাগ আগে খুলবেন, সেটি পরে বন্ধ করতে হবে।

💡 টিপস: নেস্টিং ঠিক রাখার জন্য প্রতিটি নেস্টেড লেভেলে ইন্ডেন্টেশন (ট্যাব বা স্পেস) ব্যবহার করুন। এতে কোড পড়তে ও বুঝতে সুবিধা হয়।

HTML কাঠামোর গুরুত্বপূর্ণ নিয়ম

নিয়ম বর্ণনা উদাহরণ
DOCTYPE প্রথম লাইনে <!DOCTYPE> ডিক্লারেশন সবসময় প্রথম লাইনে থাকতে হবে <!DOCTYPE html>
html এলিমেন্ট মূল <html> এলিমেন্টের ভিতরে বাকি সব এলিমেন্ট থাকবে <html>...</html>
head ও body আলাদা <head> ও <body> এলিমেন্ট আলাদা এবং সঠিক ক্রমে থাকবে head তারপর body
মেটা চারসেট বাংলা সঠিকভাবে দেখাতে <meta charset="UTF-8"> যোগ করুন <meta charset="UTF-8">
ভাষা নির্ধারণ <html> ট্যাগে lang অ্যাট্রিবিউট ব্যবহার করুন <html lang="bn">

HTML কাঠামোর সাধারণ ভুল

ভুল কী হয় সঠিক পদ্ধতি
<!DOCTYPE> না দেওয়া ব্রাউজার কোয়র্কস মোডে চলে, ডিজাইন ভুল হতে পারে সর্বদা <!DOCTYPE html> দিন
<html> ট্যাগ বাদ দেওয়া ব্রাউজার ধরে নেয়, কিন্তু ভালো অভ্যাস নয় <html> ট্যাগ ব্যবহার করুন
<head> এর ভিতরে <body> এর এলিমেন্ট লেখা এলিমেন্ট ব্রাউজারে দেখা যাবে না <head> শুধু মেটাডাটার জন্য ব্যবহার করুন
<title> ট্যাগ না দেওয়া ব্রাউজার ট্যাবে ফাইলনেম দেখায় সর্বদা <title> দিন
বাংলা এনকোডিং না দেওয়া বাংলা ক্যারেক্টার ভুল দেখায় <meta charset="UTF-8"> যোগ করুন

HTML কাঠামো চেকলিস্ট

একটি HTML পৃষ্ঠা তৈরি করার পর নিচের চেকলিস্ট ব্যবহার করে দেখতে পারেন:

  • ✅ <!DOCTYPE html> প্রথম লাইনে আছে?
  • ✅ <html> ট্যাগ আছে এবং lang অ্যাট্রিবিউট দেওয়া আছে?
  • ✅ <head> ট্যাগের ভিতরে <meta charset="UTF-8"> আছে?
  • ✅ <title> ট্যাগ আছে এবং অর্থপূর্ণ শিরোনাম দেওয়া আছে?
  • ✅ <body> ট্যাগের ভিতরে দৃশ্যমান কন্টেন্ট লেখা আছে?
  • ✅ সব ট্যাগ সঠিকভাবে বন্ধ করা হয়েছে?
  • ✅ নেস্টিং সঠিকভাবে করা হয়েছে?
  • ✅ ইন্ডেন্টেশন ব্যবহার করে কোড পড়তে সুবিধা হচ্ছে?

বিভিন্ন ধরণের HTML ডকুমেন্টের কাঠামো

একটি সাধারণ ব্লগ পোস্টের কাঠামো:

<!DOCTYPE html>
<html lang="bn">
<head>
  <meta charset="UTF-8">
  <title>আমার ব্লগ</title>
</head>
<body>
  <header>
    <h1>আমার ব্লগ</h1>
    <nav>মেনু আইটেম</nav>
  </header>
  
  <article>
    <h2>ব্লগ পোস্টের শিরোনাম</h2>
    <p>পোস্টের কন্টেন্ট...</p>
  </article>
  
  <footer>
    <p>কপিরাইট ২০২৬</p>
  </footer>
</body>
</html>

একটি যোগাযোগ পৃষ্ঠার কাঠামো:

<!DOCTYPE html>
<html lang="bn">
<head>
  <meta charset="UTF-8">
  <title>যোগাযোগ</title>
</head>
<body>
  <h1>আমাদের সাথে যোগাযোগ করুন</h1>
  
  <form>
    <label for="name">নাম:</label>
    <input type="text" id="name">
    
    <label for="email">ইমেইল:</label>
    <input type="email" id="email">
    
    <button type="submit">পাঠান</button>
  </form>
</body>
</html>

অনুশীলনী

  1. HTML ডকুমেন্টের চারটি প্রধান অংশের নাম লেখ।
  2. <head> এলিমেন্টের ভিতরে সাধারণত কী কী এলিমেন্ট থাকে? (পাঁচটি নাম লেখ)
  3. <body> এলিমেন্টের ভিতরে কী ধরনের কন্টেন্ট রাখা হয়?
  4. DOCTYPE ডিক্লারেশন না দিলে কী সমস্যা হতে পারে?
  5. বাংলা ভাষায় HTML পৃষ্ঠা তৈরি করতে কী কী অ্যাট্রিবিউট ও ট্যাগ যোগ করা প্রয়োজন?
  6. নিচের কোডে ভুল খুঁজে বের করে সঠিক কোড লেখ: 
    <html>
<head>
  <title>ভুল কাঠামো
  <body>
    <h1>শিরোনাম</h1>
  </head>
    <p>প্যারাগ্রাফ</p>
  </body>
</html>
  7. একটি HTML পৃষ্ঠা তৈরি করুন যেখানে আপনার নাম, ঠিকানা এবং প্রিয় রং থাকবে।
  8. উপরের তৈরি পৃষ্ঠায় lang অ্যাট্রিবিউট এবং meta চারসেট যোগ করে বাংলা সাপোর্ট যুক্ত করুন।
সমাধান দেখুন
  1. DOCTYPE, <html>, <head>, <body>
  2. <title>, <meta>, <link>, <style>, <script>
  3. দৃশ্যমান কন্টেন্ট যেমন: শিরোনাম, প্যারাগ্রাফ, ছবি, লিংক, তালিকা, টেবিল, ফর্ম ইত্যাদি
  4. ব্রাউজার কোয়র্কস মোডে চলে, ডিজাইন ভুল হতে পারে, HTML5 ফিচার ঠিকমত কাজ নাও করতে পারে
  5. <html lang="bn"> এবং <meta charset="UTF-8">
  6. সঠিক কোড: 
    <!DOCTYPE html>
<html>
<head>
  <title>ভুল কাঠামো</title>
</head>
<body>
  <h1>শিরোনাম</h1>
  <p>প্যারাগ্রাফ</p>
</body>
</html>

সারাংশ

HTML ডকুমেন্টের মৌলিক কাঠামো বুঝতে পারা ওয়েব ডেভেলপমেন্টের প্রথম এবং সবচেয়ে গুরুত্বপূর্ণ ধাপ। এই কাঠামো ঠিকভাবে না বুঝলে পরবর্তী জিনিসগুলো শেখা কঠিন হবে।

মনে রাখার মূল বিষয়:

  • ✅ DOCTYPE প্রথম লাইনে দিতে হবে
  • ✅ <html> সবকিছুর মূল
  • ✅ <head> এ মেটাডাটা
  • ✅ <body> এ দৃশ্যমান কন্টেন্ট
  • ✅ বাংলার জন্য lang="bn" ও meta charset="UTF-8"
  • ✅ সঠিক নেস্টিং বজায় রাখতে হবে
  • ✅ সব ট্যাগ বন্ধ করতে হবে

🎉 অভিনন্দন! আপনি এখন HTML-এর মৌলিক কাঠামো সম্পর্কে সম্পূর্ণ ধারণা পেয়েছেন। পরবর্তী অধ্যায়ে আপনি HTML ট্যাগ ও এলিমেন্ট সম্পর্কে বিস্তারিত জানবেন।
